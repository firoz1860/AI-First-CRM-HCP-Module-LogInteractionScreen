─── LANGGRAPH AGENT ────────────────────────────────────────────────────────

ACTIONS: Open `backend/agent/graph.py`

SCRIPT:
"The intelligence lives in the agent folder. Here in graph.py we define
the LangGraph StateGraph.

The state holds everything we need across a conversation: the HCP ID,
the interaction being built, extracted entities, compliance results,
and the current conversation stage.

The graph has three nodes:
  - agent_node: calls the Groq LLM with all 5 tools bound to it
  - tool_executor_node: when the LLM decides to use a tool, this fires it
  - finalize_node: cleans up and returns the response

The conditional edge logic is simple — if the last AI message has tool
calls attached, route to tool execution. Otherwise, we're done."

─── LOG INTERACTION TOOL ───────────────────────────────────────────────────

ACTIONS: Open `backend/agent/tools/log_interaction.py`

SCRIPT:
"Here's Tool 1 — log_interaction. When the agent calls this tool,
it receives the raw conversation text from the rep.

Step 1: it sends that text to Groq's Gemma2 model with a structured
extraction prompt asking for key messages, HCP feedback, products,
and a professional summary — all as JSON.

Step 2: it runs a compliance check inline.

Step 3: it inserts the enriched record into SQLite.

Step 4: it returns the interaction ID and AI summary back to the agent,
which then incorporates it into the reply to the user."

─── FRONTEND — LOG INTERACTION SCREEN ──────────────────────────────────────

ACTIONS: Open `frontend/src/features/interactions/LogInteractionScreen.jsx`

SCRIPT:
"On the frontend, LogInteractionScreen.jsx is the main component.

The mode toggle dispatches to our UI Redux slice — that's all it takes
to switch between Form and Chat mode.

The HCP selector at the top is a custom searchable dropdown that filters
the Redux hcp.list state. When an HCP is selected, it dispatches
selectHCP and fetches their recent interactions to populate the context panel.

In FormMode, when the user submits, it dispatches createInteraction which
hits POST /api/interactions/. The backend does all the AI work and returns
the enriched record — AI summary, compliance status, everything.

In ChatMode, we connect via WebSocket to the LangGraph agent. Every
incoming event — tool start, tool end, streaming tokens — is dispatched
to the chat Redux slice, which updates the UI in real time."

─── REDUX STORE ────────────────────────────────────────────────────────────

ACTIONS: Open `frontend/src/app/store.js`

SCRIPT:
"The Redux store has 5 slices:
  - interactions: the list of logged interactions plus the AI results
  - hcp: the HCP directory and the currently selected HCP
  - chat: WebSocket state, messages, streaming content, tool status
  - ui: the current mode (form or chat), sidebar state, toasts
  - dashboard: aggregated metrics data

Clean separation of concerns. Each feature only reads from its own slice."

[11:30 – 13:00]  WHAT I PERSONALLY UNDERSTOOD
================================================================================

SCRIPT (talk to camera):
"Let me share the key things I understood from building this.

The most important insight was understanding what LangGraph adds over a
plain LLM call. It's not just 'call GPT and get text back.' LangGraph
maintains typed state across multiple turns of a conversation. The agent
can call Tool 1, get a result, pass that result to Tool 5, and combine
both outputs — all in a single user message. That's impossible with
a single prompt-response cycle.

The two-model strategy made practical sense. Gemma2-9b-it is fast
and precise for structured extraction — give it a paragraph, get back
clean JSON. But Llama 3.3 70B is better for open-ended reasoning —
computing a diff, writing a recommendation, understanding complex context.
Using the right model for the right task keeps costs down and quality high.

The compliance-as-a-tool pattern is architecturally sound. Instead of
running compliance as an afterthought, it's baked into the log_interaction
tool itself. Every interaction is automatically checked. Reps can't
accidentally skip it.

And the dual-mode interface taught me something about UX in enterprise
tools: don't force users to change behavior. Some reps want a form because
it's predictable and fast. Others want to talk naturally about their visit.
Building both modes on the same Redux state means either path produces
the same structured data in the database."
